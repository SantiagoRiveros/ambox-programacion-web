¿PORQUE?
El navegador esta hecho para proteger al usuario.
Si cualquiera pudiera desde una pagina leer los archivos, seria un desastre.

Javascript TENIA una enorme limitacion, que solo podia ejecutarse en navegador.

ENTONCES... en 2009 nacio Node.js
La idea es -> Que pasa si ejecutamos JS fuera del navegador?

ANTES = JS -> Navegador
AHORA = JS -> Node.js -> Sistema Operativo/Carpetas


¿Que es node?
No es un lenguaje
No reemplaza Javascript
No es un framework

ES UN ENTORNO DE EJECUCION.

entonces -> Podemos decir que Node permite ejecutar JS fuera del navegador.

¿Que es un servidor?
Es un programa que esta esperando solicitudes.
 
Cliente -> SOLICITUD -> Servidor -> SOLICITUD -> Cliente

Frontend -> REQUEST -> Server -> RESPONSE -> Frontend

¿Como se comunican estas partes?

Mediante HTTP -> HyperText Transfer Protocol

Es el protocolo mediante el cual un cliente y un servidor se comunican.

POST, PUT, GET, DELETE.

Cuando nosotros en el navegador entramos a por ejemplo www.google.com 

Chrome -> /GET a www.google.com -> Servidor de Google -> HTML -> Chrome
